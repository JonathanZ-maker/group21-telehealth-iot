# Presentation — `/presentation`

Material for the 5-minute demo video that gets uploaded to YouTube (or any public link).

## Contents

| File | Owner | Description |
|------|-------|-------------|
| `script.md` | all | Scene-by-scene narration and shot list — aim for ≤300 spoken words per minute. |
| `demo_checklist.md` | all | List of screens/terminal windows to have open before recording. |
| `recording_notes.md` | editor | OBS / Zoom settings, resolution, mic level. |
| `youtube_link.md` | editor | The final public URL (pasted into the report cover page). |

## Suggested 5-minute structure

| Minutes | Content | Who speaks |
|---------|---------|-----------|
| 0:00–0:30 | Problem, regulation context, system overview | one member |
| 0:30–1:30 | Attack 1 live: BLE sniff / tamper / replay against undefended gateway | LKK |
| 1:30–2:30 | HMAC + AI-IDS defences live: same attacks blocked | LYZ + ZYM |
| 2:30–3:30 | Attack 2 live: NoSQL injection + unauthorised read | YYX |
| 3:30–4:30 | Pydantic + JWT defences live: injections return 400/401 | ZYM |
| 4:30–5:00 | ROC + privacy-utility plots, closing statement | all |

Keep it crisp — the marker has 5 minutes, not a second more.
